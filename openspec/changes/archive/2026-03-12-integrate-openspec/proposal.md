## Why

This repository is meant to capture durable AI workflow knowledge, so adding OpenSpec should include both the tool setup and the working conventions around it. Without that guidance, the repo would have slash commands but no shared rule for when to use them, what to commit, or how to stay human-in-the-loop.

## What Changes

- Initialize OpenSpec for `opencode` in this repository so OpenSpec slash commands and skills are available locally.
- Add repository documentation that explains how OpenSpec is installed, how it is used per project, and when it is worth using.
- Document the preferred workflow for this repo: use OpenSpec for meaningful multi-step work, commit important spec artifacts, and keep the human as reviewer and decision-maker.

## Capabilities

### New Capabilities
- `ai-workflow-conventions`: Define how this repository documents and uses OpenSpec as part of its AI workflow.

### Modified Capabilities

None.

## Impact

- Adds OpenSpec commands and skills under `.opencode/`.
- Adds OpenSpec change artifacts under `openspec/`.
- Updates repository documentation in `README.md` and `docs/playbooks/`.
